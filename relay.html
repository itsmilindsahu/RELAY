How routing works

  1. Map the mesh. Nodes within range of each other are linked, the same way phones would discover each other over Bluetooth or Wi-Fi Direct.
  2. List the candidate routes. Every reasonable path between source and destination is traced, up to 6 hops.
  3. Score by reliability, not distance. Each path is scored by the average reliability of the nodes it passes through — a longer path through solid devices can beat a short path through a flaky one.
  4. Send along the best-scoring path. Ties go to whichever route has fewer hops.
